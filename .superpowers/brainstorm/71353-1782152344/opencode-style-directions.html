C. ACP Dashboard

Still OpenCode-ish, but leans into ACP interoperability: capabilities, protocol state, server identity.

OpenCode cues to borrow

  • Near-black background, dark panels
  • Muted gray metadata
  • Orange primary, blue secondary, purple accent
  • Subtle borders, especially vertical split borders
  • Footer/status hints instead of loud chrome

Things to avoid

  • Copying branding too closely
  • Adding sidebar before we have useful data
  • Visual polish that hides protocol errors
  • Three.js effects before prompt UX works
